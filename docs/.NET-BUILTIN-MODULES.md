# .NET Built-in Modules & Features Used

## Overview

The Student Project Planner application is built entirely using Microsoft's built-in .NET modules. No external packages are used for core functionality - only standard libraries and first-party Microsoft packages.

## ASP.NET Core Framework

### Core Services & Middleware

- **Microsoft.AspNetCore.Builder** - Application builder for configuring the HTTP pipeline
- **Microsoft.AspNetCore.Hosting** - Host configuration and environment detection
- **Microsoft.AspNetCore.Routing** - URL routing and navigation
- **Microsoft.AspNetCore.StaticFiles** - Static file serving (CSS, JS, images)
- **Microsoft.AspNetCore.Http** - HTTP request/response handling
- **Microsoft.AspNetCore.Antiforgery** - CSRF token protection

### Authentication & Authorization

- **Microsoft.AspNetCore.Authentication** - Base authentication framework
- **Microsoft.AspNetCore.Authentication.Cookies** - Cookie-based authentication
- **Microsoft.AspNetCore.Authorization** - Authorization policies and attributes
- **Microsoft.AspNetCore.Identity** - User authentication, password hashing, role management
- **Microsoft.AspNetCore.Identity.EntityFrameworkCore** - Identity integration with EF Core

#### Features:

- User registration and login with email/password
- Password requirements: uppercase, lowercase, digits, minimum 6 characters
- User role management (for future authorization)
- Session management and authentication state

## Entity Framework Core 8.0

### Database Access & ORM

- **Microsoft.EntityFrameworkCore** - Core ORM framework
- **Microsoft.EntityFrameworkCore.Sqlite** - SQLite database provider (development)
- **Microsoft.EntityFrameworkCore.SqlServer** - SQL Server provider (production)
- **Microsoft.EntityFrameworkCore.Tools** - Migration and scaffold tools

#### Features:

- Code-first database design
- Automatic database migrations
- Lazy loading and eager loading of related entities
- Query optimization with LINQ to Entities
- Cascading delete and referential integrity
- Transaction support

### Database Entities

- ApplicationUser (extends IdentityUser)
- Course
- Assignment
- GroupProject
- ProjectTask
- ProjectMember
- AspNetUsers, AspNetRoles (Identity tables)

## Blazor Components & UI

### Component Framework

- **Microsoft.AspNetCore.Components** - Base Blazor component framework
- **Microsoft.AspNetCore.Components.Web** - Web-specific components (routing, layouts)
- **Microsoft.AspNetCore.Components.Forms** - Form components (EditForm, InputText, DataAnnotationsValidator)
- **Microsoft.AspNetCore.Components.Authorization** - Authentication state management
- **Microsoft.JSInterop** - JavaScript interoperability

#### Built-in Blazor Components Used:

- `CascadingAuthenticationState` - Provides auth state to child components
- `Router` - Client-side routing
- `AuthorizeRouteView` - Protected routes with authorization checks
- `Authorize` attribute - Component-level authorization
- `EditForm` - Form binding and validation
- `DataAnnotationsValidator` - Server-side validation using data annotations
- `InputText` - Text input field
- `InputDate` - Date picker
- `InputSelect` - Dropdown select
- `InputTextArea` - Multi-line text input
- `ValidationSummary` - Display validation errors
- `CascadingParameter` - Pass data down component hierarchy

## Data Validation

### System.ComponentModel.DataAnnotations

- **[Required]** - Required field validation
- **[MaxLength]** - Maximum string length
- **[Range]** - Numeric range validation
- **[EmailAddress]** - Email format validation
- **IValidatableObject** - Custom validation logic

## Dependency Injection

- **Microsoft.Extensions.DependencyInjection** - Built-in IoC container
- Service registration (Scoped, Transient, Singleton)
- Configuration-based service lookup

## Configuration Management

- **Microsoft.Extensions.Configuration** - Application configuration
- **appsettings.json** - Configuration files
- **appsettings.Development.json** - Environment-specific settings
- Connection string management
- Logging configuration

## Logging

- **Microsoft.Extensions.Logging** - Logging framework
- **Microsoft.Extensions.Logging.Console** - Console output
- Structured logging with log levels (Information, Warning, Error)

## Repository Pattern Implementation

All repositories use built-in .NET features:

- **IAsyncEnumerable<T>** - Async streaming results
- **Task<T>** - Asynchronous operations
- **LINQ** - Query composition
- **Generic interfaces** - Type-safe data access

### Repository Interfaces:

- `IRepository<T>` - Generic CRUD operations
- `ICourseRepository` - Course-specific queries
- `IAssignmentRepository` - Assignment-specific queries with filtering
- `IGroupProjectRepository` - Project management

## Service Layer Pattern

All services use built-in patterns:

- Dependency injection for repositories
- Business logic separation from UI
- Async/await for non-blocking operations
- Exception handling
- Data transfer objects (DTOs)

### Service Interfaces:

- `ICourseService` - Course management
- `IAssignmentService` - Assignment tracking
- `IGroupProjectService` - Project collaboration
- `IDashboardService` - Analytics and metrics

## HTTP Pipeline

### Middleware Configuration

```csharp
app.UseHttpsRedirection()      // HTTPS enforcement
app.UseStaticFiles()           // Static file serving
app.UseRouting()               // Routing
app.UseAuthentication()        // Authentication
app.UseAuthorization()         // Authorization
app.UseAntiforgery()          // CSRF protection
```

## Page Routes

All routes use standard Blazor routing:

- `/` → Dashboard.razor
- `/courses` → Courses.razor
- `/assignments` → Assignments.razor
- `/projects` → Projects.razor
- `/login` → Login.razor
- `/register` → Register.razor
- `/logout` → Logout.razor

## Security Features (Built-in)

- **HTTPS enforcement** - Automatic redirect to HTTPS
- **CSRF protection** - Antiforgery tokens
- **Password hashing** - ASP.NET Core Identity (PBKDF2)
- **Session security** - HttpOnly cookie flags
- **Authorization attributes** - Role and policy-based access control
- **SQL injection prevention** - Parameterized queries through EF Core

## NuGet Packages Used

All packages are Microsoft first-party packages:

- **Microsoft.AspNetCore.Identity.EntityFrameworkCore** - v8.0.0
- **Microsoft.EntityFrameworkCore.SqlServer** - v8.0.0
- **Microsoft.EntityFrameworkCore.Sqlite** - v8.0.0

Test packages (xUnit, Moq, FluentAssertions) are optional and not used by the main application.

## Conclusion

This application demonstrates best practices for building modern .NET web applications using only built-in Microsoft technologies:

- Clean architecture with repository and service patterns
- Entity Framework Core for data persistence
- ASP.NET Core Identity for authentication
- Blazor for interactive UI components
- Built-in dependency injection for loose coupling
- Standard middleware pipeline for HTTP handling
- Security best practices with HTTPS and CSRF protection

All functionality required for the Student Project Planner is implemented using only these built-in modules.
